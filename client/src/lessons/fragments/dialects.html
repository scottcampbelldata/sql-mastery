SQL Dialects: Postgres vs. BigQuery vs. Snowflake

The logic you have learned here is standard SQL and ports everywhere. What changes between warehouses is the surface: date functions, string splitting, casting, and a few conveniences. This is the reference to skim before a BI interview - your engine here is PostgreSQL, so the Postgres column is what you have been writing; the other two are the ones you will be asked to "also know." When unsure in an interview, say the standard-SQL form and note the dialect difference out loud.

Dates and times

TaskPostgreSQLBigQuerySnowflake
TodayCURRENT_DATECURRENT_DATE()CURRENT_DATE
Truncate to monthdate_trunc('month', ts)DATE_TRUNC(dt, MONTH)DATE_TRUNC('month', ts)
Extract a partEXTRACT(YEAR FROM ts)EXTRACT(YEAR FROM ts)YEAR(ts) / EXTRACT(...)
Add an intervalts + INTERVAL '7 days'DATE_ADD(dt, INTERVAL 7 DAY)DATEADD('day', 7, ts)
Difference in daysd2 - d1DATE_DIFF(d2, d1, DAY)DATEDIFF('day', d1, d2)
Format for displayTO_CHAR(ts, 'YYYY-MM')FORMAT_DATE('%Y-%m', dt)TO_CHAR(ts, 'YYYY-MM')
The one that trips people up Date subtraction returns days in Postgres (d2 - d1) but you must spell out the unit in BigQuery and Snowflake. And BigQuery's DATE_TRUNC takes the column first and an unquoted unit (DATE_TRUNC(dt, MONTH)), the reverse of Postgres. If you only remember one dialect gotcha, remember date functions - they differ the most.

Analytics conveniences

TaskPostgreSQLBigQuerySnowflake
Top-1 per groupROW_NUMBER in a subquery, filter = 1QUALIFY ROW_NUMBER() OVER (...) = 1QUALIFY ROW_NUMBER() OVER (...) = 1
Semi-structuredjsonb, -> / ->>; native arraysJSON, JSON_VALUE; ARRAY + UNNESTVARIANT, col:path; FLATTEN
Approx distinct count(extension)APPROX_COUNT_DISTINCT(x)APPROX_COUNT_DISTINCT(x)
QUALIFY is the flex QUALIFY filters on a window function without wrapping the query in a subquery - SELECT ... QUALIFY ROW_NUMBER() OVER (PARTITION BY user ORDER BY ts DESC) = 1 is the whole "latest row per user" query. Postgres has no QUALIFY, so you write the CTE-plus-filter form you already know. Mentioning that QUALIFY is a BigQuery/Snowflake shortcut for exactly that pattern signals you have worked across warehouses.

How to use this

Do not memorize it cold. Learn the shape of each operation in Postgres here - split-then-explode, truncate-then-group, guard-with-NULLIF - and keep this page for the day you switch dialects. Interview move: write the standard form, and when a function name might differ, say "in BigQuery that's SPLIT and UNNEST; in Snowflake it's FLATTEN" - that one sentence turns a syntax gap into a demonstration of range.
